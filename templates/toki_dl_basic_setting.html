뉴토끼 다운로더 — 사용법

스케줄러가 돌 때마다 설정에 적힌 작품들을 순회 (만화/웹툰/소설 분리):

  1. 작품 페이지 ({base_url}/{kind}/{workId}) SSR HTML 파싱
    • 회차 목록 + 작품 메타 추출
    • info.xml / cover.jpg 자동 생성 (없으면)
  2. 회차 분류
    • 무료 → 다운로드 대상
    • 유료 (paid-gate) → 자동 스킵
  3. 미수신 회차 중 최신 N개 (max_per_run) 만 다운로드
  4. 만화/웹툰: 회차당 이미지 N장 → {경로}/{kind}/{작품}/{NNNN_회차}/{001.jpg ...}
  5. 소설: 회차당 본문 1개 → {경로}/novel/{작품}/{NNNN_회차}.txt
  6. DB(toki_dl_item) 에 회차별 이력 기록 — 같은 회차 재다운로드 안 함

옵션

  • 회차 폴더 ZIP 압축 (만화/웹툰만, 소설은 .txt 그대로)
  • 다운 완료 시 Discord/Slack 웹훅 알림 (만화/웹툰 / 소설 별도 채널)
  • 도메인 / 쿠키 만료 시 알림 웹훅 (인증 탭)
  • 도메인 자동 갱신 — 호스트 끝 숫자 +1~+3 증가 시도

핵심 특징

  • 무로그인 — 별도 회원가입/쿠키 입력 없이 작동. 일부 환경에서만 인증 탭의 쿠키 입력 필요.
  • 검색 없음 — 입력은 URL 또는 workId 만. 동명 작품 너무 많아 검색 신뢰 불가.
  • 3종 분리 — 만화/웹툰/소설 각각 별도 textarea + 폴더.
  • 도메인 로테이션 대응 — 뉴토끼는 도메인을 자주 바꿈. 인증 탭의 base_url 갱신 또는 자동 갱신.
  • 소설 본문 지원 — 텍스트 뿐 아니라 HTML/CSS 포함 회차도 일반 텍스트로 변환 저장.

문제 해결

  • "접속 실패" 알림이 오면 → 인증 탭의 뉴토끼 도메인 을 안내 사이트 링크에서 확인 후 수동 갱신.
  • 다운로드 중 차단(blocked) 메시지가 자주 뜨면 → 인증 탭에 브라우저 쿠키 입력(특히 cf_clearance) 또는 프록시 설정.
  • 소설 다운로드만 작동하지 않으면 → 본 빌드에 소설 모듈이 포함되지 않은 경우. 만화/웹툰은 정상 작동.
  • 이력 초기화는 일반 탭의 "이력 초기화" 버튼으로. 작품 삭제는 textarea 에서 줄 제거.
